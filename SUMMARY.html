Campus Systems Architecture Summary

1. Design Philosophy

Given the strict real-time constraints (latency over throughput) and the 2.00x bandwidth budget, standard ARQ (Automatic Repeat reQuest) protocols were rejected due to the inherent Round-Trip-Time (RTT) penalty. Forward Error Correction (FEC) block-parity was rejected due to forced sender-side delay. The optimal solution was a stateless proactive redundancy model paired with a strictly-timed receiver jitter buffer.

2. Sender Architecture (N-1 Piggybacking)

The sender reads the 164-byte frame from the harness and caches the payload. It constructs a custom 324-byte UDP datagram containing the 4-byte sequence number, the current 160-byte payload, and the cached 160-byte payload from the previous frame. If packet i is dropped, packet i+1 seamlessly delivers the missing data with zero latency penalty.

The 1/30 Bandwidth Exploit: A strict 324-byte packet results in a 2.02x bandwidth overhead, failing the hard constraint. To solve this, the sender omits the redundant payload on seq % 30 == 0. This micro-optimization reduces total bandwidth to 1.99x while maintaining 96.6% redundancy coverage.

3. Receiver Architecture (Clock-Priority Event Loop)

The receiver pre-allocates a 1024-slot circular std::array jitter buffer. It operates on a two-phase event loop to prevent starvation:

  • Phase 1 (Network Drain): Uses poll() with a 0ms timeout to aggressively drain the OS socket buffer, reconstructing missing frames on the fly.
  • Phase 2 (Strict Playout): Compares absolute epoch time against the calculated deadline (T0 + DELAY_MS + i * 20ms - 2ms buffer). Frames are delivered to the player exactly when due, completely decoupling network arrival time from playout time.

4. Limitations

This architecture trades structural complexity for speed. It cannot survive burst drops of length 2 or greater. It mitigates this limitation by leveraging a higher DELAY_MS to absorb all jitter misses, leaving the entire 1.00% error budget available to absorb inevitable burst drops.
